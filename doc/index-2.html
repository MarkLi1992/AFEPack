2. Installation

2.1 How to obtain AFEPack

The package can be obtained from the homepage of the software http://www.ukc.ac.uk/cbs/staff/homepage/wbl/data/AFELAB-AFEPACK.htm. The users in P.R.China perhaps feel more convenient to download it from http://circus.math.pku.edu.cn/index.html/source_code/AFEPack.tar.gz.

2.2 Requirements

  • The standard template library is needed at first.
  • The package is based on the linear algebric library of the deal.II library. So you should have the head file and library of the base part and lac part of deal.II.
  • The package adopt dynamic library operation, so you may have the C library development package installed, such as the glibc-devel package on Linux system.
Now AFEPack can't work on HP-UX because the uncompatibility in shared library loading from HP-UX to other systems. Note: in fact the package before version 1.5 is only tested on Linux system.

2.3 Compilation and installation

In order to compile and install AFEPack on your system, you need

  1. Download the AFEPack tar package file;
  2. Uncompress the package in certain directory you like, saying "/usr/local/AFEPack", as
% cd /usr/local
% gzip -d AFEPack-1.2.tar.gz
% tar xvf AFEPack-1.2.tar
% cd AFEPack
  1. Compile the library as
% ./configure
% make
  1. Build symbolic links for the library file and the head file to where you want to keep them as
% ln -s /usr/local/AFEPack/library/include /usr/local/include/AFEPack
% ln -s /usr/local/AFEPack/library/lib/lib* /usr/local/lib
  1. Set environment variable. AFEPack using an environment variable to tell the library where to find those shared library and information of those template element, that is "AFEPACK_TEMPLATE_PATH".  We assume that in your application, you will use triangle and rectangle elements, then you can set (in bash)
% export AFEPACK_TEMPLATE_PATH=/usr/local/AFEPack/template/triangle:/usr/local/AFEPack/template/rectangle
  1. Write your own application and link it with the library.
Since AFEPack uses GNU build tool set(except for Libtool), you should have no trouble compiling it. Should you run into problems please report them to the the author at Robert Lie .
